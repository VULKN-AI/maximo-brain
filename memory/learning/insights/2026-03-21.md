evidence: "March 21 session hit 90% context (180k/200k tokens), triggering compaction #1. The pre-compaction memory flush captured state into MEMORY.md, revealing 2 previously uncaptured items: 'Frida el Musical' (5th active event, April 11 in Acapulco) and ZENON's ticket issue (paid 15 days ago, never received tickets). stakes: medium
domain: operations
context: "Long-running sessions with customer interactions accumulate knowledge that may not yet be written to persistent files. Compaction discards context. The flush bridges this gap."
behavioral_change: "Always write accumulated knowledge to persistent files before context exceeds 70%, not just at compaction. Treat MEMORY.md updates as transactional checkpoints during busy sessions."
alternatives:
  - "Could rely on nightly consolidation to capture everything (risk: data loss between sessions)"
  - "Could reduce context window to force more frequent compactions and flushes"
